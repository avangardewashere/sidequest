# Chapter 2 - Retention Sprint (Daily Quests + Milestones)

This chapter documents the retention-focused sprint added after the core Chapter 1 gameplay loop.

## 1) Sprint objective

Increase repeat usage by introducing:
- Daily quests that refresh each UTC day
- Milestone streak rewards for consistency
- UI feedback that makes milestone progress visible
- Metrics for tuning daily and reward behavior

## 2) What was implemented

## Daily quest system

- Added daily metadata to quests:
  - `isDaily` (boolean)
  - `dailyKey` (string key like `YYYY-MM-DD` in UTC)
- Added deterministic daily generation:
  - Same user + same day -> same set of daily quests
  - 3 daily quests generated only if none exist for that day
- New endpoint:
  - `GET /api/dailies`

Files:
- `src/models/Quest.ts`
- `src/lib/dailies.ts`
- `src/app/api/dailies/route.ts`

## Streak milestone rewards

- Added milestone reward logging model to prevent duplicate rewards.
- Milestone thresholds:
  - 3-day streak -> +15 bonus XP
  - 7-day streak -> +40 bonus XP
  - 14-day streak -> +100 bonus XP
- Quest completion endpoint now:
  - checks if streak hit a milestone
  - grants bonus XP once per milestone
  - returns `milestoneReward` metadata in API response

Files:
- `src/models/MilestoneRewardLog.ts`
- `src/lib/progression.ts`
- `src/app/api/quests/[id]/complete/route.ts`

## Dashboard updates

- Dashboard now fetches and displays daily quests.
- Completing a quest shows either:
  - normal XP message
  - or milestone reward message including bonus XP

File:
- `src/app/page.tsx`

## Metrics expansion

`GET /api/metrics/summary` now includes:
- `dailyQuestsCreated`
- `dailyQuestsCompleted`
- `dailyCompletionRate`
- `milestoneRewardsTriggered`

File:
- `src/app/api/metrics/summary/route.ts`

## 3) Schema additions explained

## Quest schema additions

- `isDaily`: marks quests generated by the daily system
- `dailyKey`: identifies the UTC day the daily belongs to
- Index added for daily lookup:
  - `{ createdBy: 1, isDaily: 1, dailyKey: 1 }`

Why:
- Fast fetch of todayâ€™s daily quests per user
- Clean separation of normal quests vs daily quests

## MilestoneRewardLog schema

Fields:
- `userId`
- `streakMilestone`
- `bonusXp`
- `awardedAt`

Indexes:
- Unique index on `{ userId, streakMilestone }` to enforce one-time reward per threshold
- Time-oriented index on `{ userId, awardedAt }` for analytics

Why:
- Prevents duplicate milestone payouts
- Enables milestone telemetry

## 4) Vital functions introduced/extended

## `src/lib/dailies.ts`

- `getUtcDailyKey(date?)`
  - Returns stable UTC day key format `YYYY-MM-DD`
- `buildDailyQuestSet(userId, dailyKey)`
  - Deterministically picks 3 templates for the user/day
  - Sets difficulty and server-derived XP reward

## `src/lib/progression.ts`

- `STREAK_MILESTONE_BONUS`
  - Central map of streak threshold to XP bonus
- `getMilestoneBonus(streak)`
  - Returns bonus XP or `null` when streak has no milestone

## 5) Core data flow

1. Dashboard calls `GET /api/dailies`.
2. API finds todayâ€™s dailies by `createdBy + isDaily + dailyKey`.
3. If absent, API generates and stores 3 daily quests.
4. User completes a quest via `PATCH /api/quests/:id/complete`.
5. Completion updates base progression, then checks milestone reward.
6. If milestone hit and not logged before, grant bonus XP + create reward log.
7. API returns normal completion data plus optional `milestoneReward`.

## 6) Operational notes

- Daily keys are UTC-based for consistency across servers.
- Milestone uniqueness is enforced in the DB schema, not just code logic.
- Completion flow is still sequential writes; at higher scale, migrate to DB transactions for stronger consistency.
